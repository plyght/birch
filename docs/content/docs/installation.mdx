Birch can be installed in multiple ways depending on your platform and preferences.

## From Source

Build from source using Cargo:

```bash
git clone https://github.com/plyght/birch
cd birch
cargo build --release
sudo cp target/release/birch /usr/local/bin/
```

## Pre-built Binaries

Download pre-built binaries from the [releases page](https://github.com/plyght/birch/releases):

```bash
# Linux
wget https://github.com/plyght/birch/releases/latest/download/birch-linux-x86_64
chmod +x birch-linux-x86_64
sudo mv birch-linux-x86_64 /usr/local/bin/birch

# macOS
wget https://github.com/plyght/birch/releases/latest/download/birch-darwin-x86_64
chmod +x birch-darwin-x86_64
sudo mv birch-darwin-x86_64 /usr/local/bin/birch

# Windows
# Download birch-windows-x86_64.exe and add to PATH
```

## Docker

Run Birch in a container:

```bash
docker pull birch:latest

# Run a command
docker run --rm birch:latest --help

# Run with environment variables
docker run --rm \
  -e VERCEL_TOKEN=your-token \
  -e VERCEL_PROJECT_ID=your-project \
  birch:latest rotate MY_API_KEY --env prod --service vercel
```

## Using Make

If you have the repository cloned:

```bash
make release    # Build release binary
make install    # Install to /usr/local/bin
make docker     # Build Docker image
```

## Verify Installation

Check that Birch is installed correctly:

```bash
birch --version
birch --help
```

You should see the version number and help text.
